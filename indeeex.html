🎯 Propósito del portafolio

Este portafolio digital explora, analiza y aplica los diferentes modelos de evaluación de calidad a Recursos Educativos Digitales (RED), con el fin de identificar fortalezas, limitaciones y adaptaciones al contexto del área de Tecnología e Informática. A lo largo del sitio se presentan seis modelos de evaluación, se seleccionan tres para su aplicación práctica, se evalúan dos recursos educativos digitales y se propone un modelo adaptado considerando aspectos como conectividad, accesibilidad, pertinencia curricular y mediación pedagógica.

📋 Modelos de evaluación de recursos educativos digitales

Se presentan seis modelos de evaluación de calidad de RED: CODA, LORI, MERLOT, UNE 71362, EVALUAREED y Reeves. Cada modelo aporta una mirada distinta sobre la calidad didáctica, tecnológica, usabilidad y efectividad pedagógica.

🔹 1. CODA (Calidad de Objetos de Aprendizaje)

Descripción: Combina criterios pedagógicos y tecnológicos. Evalúa coherencia didáctica, contenidos, interactividad, motivación, diseño, usabilidad, accesibilidad, reusabilidad e interoperabilidad.

Métrica: Escala 1 a 5. Instrumento: Rúbrica o formulario CODA.

🔹 2. LORI (Learning Object Review Instrument)

Descripción: Evalúa calidad del contenido, alineación con objetivos, retroalimentación, motivación, diseño, interacción, accesibilidad, reusabilidad y estándares.

Métrica: Escala 1 a 5. Instrumento: Matriz LORI.

🔹 3. MERLOT Peer Review (Modelo de evaluación)

Descripción: Modelo práctico centrado en tres criterios: calidad del contenido, efectividad como herramienta de enseñanza y facilidad de uso.

Métrica: Escala 1 a 5. Instrumento: Ficha de revisión MERLOT.

Nota: Es independiente del repositorio homónimo; se describe como herramienta de evaluación.

🔹 4. UNE 71362

Descripción: Marco amplio para valorar calidad de materiales educativos digitales: descripción didáctica, contenidos, navegación, accesibilidad, interactividad, portabilidad, etc.

🔹 5. EVALUAREED

Descripción: Evalúa recursos electrónicos en nueve dimensiones: contenido, objetivos, feedback, usabilidad, motivación, accesibilidad, requerimientos técnicos, propiedad intelectual y efectividad del aprendizaje.

🔹 6. Reeves (Dimensiones pedagógicas)

Descripción: Se enfoca en el enfoque pedagógico: epistemología, filosofía pedagógica, rol del docente, control del estudiante, aprendizaje cooperativo, sensibilidad cultural, etc.

📁 Recursos Educativos Digitales seleccionados

🌐 RED 1: Introducción a HTML/CSS (Khan Academy)

Área: Tecnología e Informática | Nivel: Básica secundaria y media

Autor: Khan Academy | Enlace: Acceder al RED

Descripción: Unidad interactiva para aprender HTML y CSS mediante videos, ejercicios y retos prácticos. Permite crear páginas web reales.

Características: Interactivo, progresivo, gratuito, con insignias de logro. Limitaciones: Requiere conexión a Internet; algunos videos sin subtítulos en español.

Estándares: Usabilidad alta, accesibilidad básica, licencia CC BY-NC-SA.

🖼️ [Captura de pantalla de Khan Academy HTML/CSS]

🎮 RED 2: Blockly Games

Área: Pensamiento computacional | Nivel: Grados 4° a 8°

Autor: Google Blockly | Enlace: blockly.games

Descripción: Colección de juegos para aprender programación por bloques: secuencias, ciclos, condicionales y funciones.

Características: Gamificado, motivador, retroalimentación inmediata. Limitaciones: Requiere conexión, falta texto alternativo en algunas imágenes.

🔄 Rediseño pedagógico de los recursos

📝 Proyecto: "Mi primera página web para comunicar una idea" (Khan Academy)

Se propone un proyecto donde los estudiantes construyen una página web con sentido comunicativo (campaña ambiental, biografía, recetario, etc.).

Guía descargable incluye: objetivo, boceto en papel, estructura HTML básica, estilos CSS, rúbrica de evaluación.

📌 Bitácora del programador (Blockly Games)

Los estudiantes registran niveles superados, estrategias usadas, errores y soluciones. Fomenta la reflexión metacognitiva.

📴 Actividades desconectadas

  • HTML/CSS: Mapa de etiquetas en papel, diseño manual de una página.
  • Blockly: Bloques recortables para ordenar secuencias físicamente antes de programar.

Mejoras transversales: guías de inicio con lenguaje claro, rúbricas formativas, trabajo colaborativo, evidencias de aprendizaje (capturas, bitácoras).

Conclusión: Khan Academy es más adecuado para secuencias completas de diseño web; Blockly Games es ideal para iniciación al pensamiento computacional. El modelo CODA adaptado es el más pertinente para el contexto escolar.

📋 Resultados LORI y MERLOT (resumen)

LORI: Khan Academy 4.33/5, Blockly Games 4.00/5. Destaca motivación en Blockly y calidad de contenido en Khan Academy.

MERLOT: Khan Academy 4.67/5, Blockly Games 4.00/5. Ambos efectivos pero Khan Academy superior en estructura didáctica.
